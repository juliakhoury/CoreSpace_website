Team Project

Project Showcase

CoreSpace is a real-time multiplayer 2D virtual environment developed using Unity and Photon PUN2, designed to redefine online collaboration through interactive and immersive experiences. Users are placed in a shared digital space where they can navigate freely, communicate via integrated voice chat, and create or join collaborative rooms with ease. The platform is further enhanced by an embedded AI assistant capable of supporting users with tasks such as answering questions, summarizing content, and facilitating learning in real time. By combining game mechanics with intelligent systems, CoreSpace transforms traditional remote work and study into a more engaging and dynamic experience

02 — Features
What We Built
🌐

Real-Time Multiplayer

Built using Photon PUN2 to synchronize player movement and interactions across all connected clients, ensuring a consistent and real-time shared environment.

🎙

Voice Chat

Integrated real-time voice communication using Photon Voice, allowing users to interact naturally within the virtual environment without relying on external platforms.

🔒

Private & Public Rooms

Supports both public and private collaboration spaces with dynamically generated access codes, providing flexibility and control over user interactions.

🚪

Portal Meetings

Walk into a portal to instantly create or join a meeting room with a randomly generated code.

🧠

AI Chat Bot

The AI system operates through a dedicated Python backend connected to a local language model. It leverages a Retrieval-Augmented Generation (RAG) pipeline to retrieve relevant contextual data before generating responses. This architecture improves response accuracy while maintaining efficient real-time performance within the game environment.

05 — Challenges
Problems We Solved

Player Ownership & Control Bugs

Remote players' prefabs were being controlled locally, causing erratic movement for all clients.

Switched TopDownMover to MonoBehaviourPun with photonView.IsMine checks to restrict input to the owning client only.

Race Conditions on Room Leave

Players leaving a room caused null reference errors due to destroyed prefabs still being accessed by other scripts.

Implemented coroutine-based delays before scene transitions, ensuring cleanup completes before the player object is destroyed.

Lobby Exit Errors with Voice

Photon Voice threw exceptions when transitioning from lobby back to menu because the voice client was still connected.

Called PhotonNetwork.Disconnect() before loading the menu scene, allowing both PUN and Voice clients to shut down cleanly.

Room List Not Populating

The public room browser showed an empty list even when rooms existed on the server.

Fixed callback subscription and ensured the client was in the lobby before requesting room list updates via OnRoomListUpdate.

AI Integration

Integrating real-time AI communication introduced latency and synchronization challenges between the game and backend.

Developed a structured request-response pipeline between Unity and a Python backend to handle AI queries efficiently.

RAG Engine

TBase AI responses lacked contextual accuracy and relevance.

Implemented a Retrieval-Augmented Generation (RAG) system to enhance responses using external context retrieval.
